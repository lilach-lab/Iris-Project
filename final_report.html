Iris Classification Project: Detailed Process Report

🚀 Project Outcome: 95.00% Accuracy

This report provides an exhaustive overview of the data science lifecycle for the Iris classification task. The final machine, built using a neural network, achieved a precision-level of 95.00% across four distinct categories.

📊 The Iterative Path to 95.00%

Achieving a 95.00% accuracy was an evolutionary process. The "Classification Machine" underwent several generations of refinement:

  • Baseline (87.50%): Our initial attempt using a simple Decision Tree and KNN ($k=3$) provided a solid foundation but struggled to distinguish the synthetic "No-Iris" noise from real botanical data.
  • Optimization (90.00%): By implementing Feature Normalization and performing Hyperparameter Tuning (optimizing to $k=5$), we increased the model's precision, but the boundary between Virginica and No-Iris remained blurry.
  • Breakthrough (95.00%): The final leap was achieved by switching to a Neural Network (Synaptic MLP). By training the machine over 20,000 iterations of backpropagation, the model successfully "learned" the non-linear patterns required to isolate the synthetic noise, reaching our peak performance of 95.00%.

Phase 1: Data Architecture & Acquisition

1.1 Dataset Composition

The project utilized a hybrid dataset combining biological and synthetic data:

  • Original Iris Dataset (150 Samples): Three botanical species—Setosa, Versicolor, and Virginica.
  • Custom "No-Iris" Class (50 Samples): Synthetic noise data points generated randomly within the feature ranges to challenge model robustness.

1.2 Feature Detailed Description

The machine uses four numerical features for every observation:

  1. Sepal Length (cm): Length of the outer flower leaves.
  2. Sepal Width (cm): Width of the outer flower leaves.
  3. Petal Length (cm): Length of the inner flower leaves.
  4. Petal Width (cm): Width of the inner flower leaves.

Phase 2: Preprocessing & Data Pipeline

2.1 Feature Normalization

To ensure numerical stability during training, every feature was normalized to a 0.0 to 1.0 scale using the formula:
X_norm = (X - X_min) / (X_max - X_min)

2.2 80/20 Data Partitioning

We implemented a strict "hold-out" evaluation strategy:

  • Training Set (80% / 160 Samples): Used exclusively for the backpropagation learning process.
  • Testing Set (20% / 40 Samples): Hidden from the model during training to provide an unbiased accuracy check.

Phase 3: The Machine (Neural Network Architecture)

The "Classification Machine" is a Multi-Layer Perceptron (MLP) with the following layers:

  • Input Layer: 4 neurons (one for each feature).
  • Hidden Layer: 10 neurons (captures complex patterns).
  • Output Layer: 4 neurons (one for each category: Setosa, Versicolor, Virginica, No-Iris).
